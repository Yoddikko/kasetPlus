# Security Policy

## Reporting a Vulnerability

We take the security of KasetPlus seriously. If you discover a security vulnerability, please report it to us responsibly and confidentially.

### How to Report

**Please DO NOT** open a public issue for security vulnerabilities. Instead:

1. **Email us directly** at: alessioiodiceuni@gmail.com
2. **Include in your report:**
   - Description of the vulnerability
   - Steps to reproduce (if applicable)
   - Potential impact
   - Your contact information (email, and optionally your GitHub username)

### What to Expect

- **Initial Response:** We will acknowledge your report within 48 hours
- **Investigation:** We will investigate and validate the vulnerability
- **Fix Development:** We will work on a fix as quickly as possible
- **Coordinated Disclosure:** We will coordinate with you on the timing of the fix and public disclosure
- **Credit:** We will credit you in the security advisory (unless you prefer to remain anonymous)

## Security Considerations

### Known Limitations

This is a fork of [kaset](https://github.com/sozercan/kaset) with additional features including:
- AdBlocker
- SponsorBlocker
- Return YouTube Dislike integration
- Other browser/app extensions

### Scope

This security policy applies to:
- The KasetPlus macOS application
- All code and dependencies in this repository
- Official releases available on this GitHub repository

## Supported Versions

| Version | Supported          |
|---------|-------------------|
| Latest  | ✅ Yes            |
| Older   | ⚠️ Case by case   |

We recommend always using the latest version of KasetPlus for the best security updates.

## Security Best Practices for Users

When using KasetPlus:

- 🔄 Keep the app updated to the latest version
- 🔐 Use strong passwords for your YouTube account
- ⚠️ Be cautious with third-party integrations
- 📢 Report any suspicious behavior to us

## Transparency

- We will disclose security vulnerabilities through GitHub Security Advisories
- CVE assignments will be requested when appropriate
- We commit to transparent communication with our users

## Questions or Concerns?

If you have questions about this security policy, please reach out to the maintainers through GitHub Discussions.

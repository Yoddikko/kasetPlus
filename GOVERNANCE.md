# Governance

## Project Leadership

KasetPlus is maintained by **Yoddikko** as the primary maintainer and decision-maker for the project.

This is an active fork of [kaset](https://github.com/sozercan/kaset) with additional features and improvements.

## Roles and Responsibilities

### Maintainer (Yoddikko)

- **Decision Authority**: Final say on all project decisions
- **Code Review**: Approves or requests changes on pull requests
- **Releases**: Manages version releases and changelog
- **Community Management**: Responds to issues and discussions
- **Project Direction**: Sets priorities and roadmap
- **Moderation**: Enforces the Code of Conduct

### Contributors

- **Code Contributions**: Submit pull requests with improvements
- **Bug Reports**: Report issues and vulnerabilities
- **Feature Requests**: Suggest new features via discussions
- **Community Support**: Help others in discussions and issues

### Community Members

- **Feedback**: Use the app and provide feedback
- **Bug Reports**: Report issues you encounter
- **Ideas**: Share feature ideas and suggestions
- **Engagement**: Participate in discussions

## Decision-Making Process

### Minor Changes
- Bug fixes, documentation, small improvements
- **Decision**: Maintainer approves directly
- **Process**: Simple code review

### Major Changes
- New features, breaking changes, significant refactors
- **Decision**: Maintainer considers community feedback
- **Process**: Discussion in GitHub Discussions or Issues before implementation

### Critical Decisions
- Project direction, major architecture changes, policy updates
- **Decision**: Maintainer makes final decision
- **Process**: Community input through Discussions strongly encouraged

## Contributing Guidelines

1. **Fork and Branch**: Create a feature branch from `main`
2. **Code Quality**: Follow project standards and existing code style
3. **Testing**: Test your changes thoroughly
4. **Documentation**: Update docs if needed
5. **Pull Request**: Submit PR with clear description
6. **Review**: Respond to feedback and make adjustments
7. **Merge**: Maintainer merges when ready

See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed guidelines (if available).

## Conflict Resolution

If there are disagreements or conflicts:

1. **Discussion**: Use GitHub Discussions or Issues for civil discussion
2. **Escalation**: Reach out to the maintainer directly if needed
3. **Code of Conduct**: All community members must follow the Code of Conduct
4. **Final Decision**: Maintainer has final authority on disputes

## Code of Conduct

All community members must adhere to the [Code of Conduct](CODE_OF_CONDUCT.md). Violations will be addressed by the maintainer.
